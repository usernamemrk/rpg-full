# Fundação Visual — Implementation Spec

> **For agentic workers:** Use superpowers:subagent-driven-development to implement task-by-task.

**Goal:** Make the core RPG experience fully functional: map editor renders tiles, ambient sounds play, inventory is viewable in-game, and spell effects fire correctly.

**Architecture:** Four independent fixes/features that share no state. Each can be implemented and tested in isolation.

**Tech Stack:** React + TypeScript + Canvas 2D + Web Audio API (client), Express + Prisma + PostgreSQL (server)

## Problem 1 — Map Renders Nothing

### Root Cause
- `TileSet.ts` only draws when `this.image` is loaded. Image loads from `/assets/tilesets/world.png` which does not exist.
- All new maps have every tile = `0`. `TileMap.render` skips `tile <= 0`. Even if the image loaded, nothing appears on a fresh map.

### Fix — `TileSet.ts`

Change `drawTile` guard and add fallback:

```ts
drawTile(ctx: CanvasRenderingContext2D, index: number, dx: number, dy: number, tileSize: number) {
  // tile 0 = empty, never draw
  if (index <= 0) return
  if (this.image) {
    const sx = ((index - 1) % this.tilesPerRow) * tileSize
    const sy = Math.floor((index - 1) / this.tilesPerRow) * tileSize
    ctx.drawImage(this.image, sx, sy, tileSize, tileSize, dx, dy, tileSize, tileSize)
    return
  }
  // Procedural fallback: colored rect when no image loaded
  const hue = (index * 37) % 360
  ctx.fillStyle = `hsl(${hue}, 55%, 38%)`
  ctx.fillRect(dx, dy, tileSize, tileSize)
  ctx.strokeStyle = `hsl(${hue}, 55%, 28%)`
  ctx.lineWidth = 1
  ctx.strokeRect(dx + 0.5, dy + 0.5, tileSize - 1, tileSize - 1)
  ctx.fillStyle = 'rgba(255,255,255,0.5)'
  ctx.font = `${Math.floor(tileSize * 0.35)}px monospace`
  ctx.textAlign = 'center'
  ctx.textBaseline = 'middle'
  ctx.fillText(String(index), dx + tileSize / 2, dy + tileSize / 2)
}
```

### Fix — `MapEditor.tsx`

Change palette tile rendering to use the **same** hue formula (`index = i + 1`, same multiplier 37, same saturation/lightness) so palette and canvas always match:

```tsx
// was: hsl(${(i * 40) % 360}, 60%, 40%)
// now:
style={{ background: `hsl(${((i + 1) * 37) % 360}, 55%, 38%)` }}
```

## Problem 2 — Ambient Sounds Silent

### Root Cause
1. `loadAmbient` tries `fetch('/assets/audio/{name}.ogg')` then `.mp3` — neither exists, so `buffers` stays empty.
2. `crossfadeTo` exits early when `buffers.get(name)` is undefined.
3. Compounding: `AudioContext` created lazily by `get context()`. If `crossfadeTo` is the first call, the context may be in `suspended` state due to browser autoplay policy. `resume()` must be awaited **before** `createBufferSource().start()`.

### Fix — `AudioManager.ts`

In `loadAmbient`, after the fetch loop fails, call `this.generateProcedural(name)` which:
1. Creates an `OfflineAudioContext(1, sampleRate * 4, sampleRate)` (4 s mono, 44100 Hz) to render offline — avoids autoplay policy during generation.
2. Connects oscillator/noise nodes to the offline context destination.
3. Calls `offlineCtx.startRendering()` and stores the resulting `AudioBuffer` in `this.buffers`.

Sound profiles per ambient (all looped 4 s buffers):

| Ambient | Oscillator | Freq | Filter | LFO |
|---------|-----------|------|--------|-----|
| floresta | noise | — | LPF 600 Hz | tremolo 0.3 Hz |
| caverna | sine | 55 Hz | — | tremolo 0.1 Hz |
| cidade | noise | — | BPF 600 Hz Q=1 | — |
| dungeon | sawtooth | 80 Hz | LPF 200 Hz | tremolo 0.05 Hz |
| batalha | square | 110 Hz | — | AM 4 Hz |
| chuva | noise | — | LPF 1200 Hz | — |

Noise is generated by filling a buffer with `Math.random() * 2 - 1`.

**`resume()` guarantee:** `crossfadeTo` must call `await this.resume()` (make `resume` async, `await ctx.resume()`) before creating the new source, even if context state appears `running`. This covers browsers that suspend between user gestures.

## Problem 3 — Inventory Has No UI

### Current State
`InventoryItem` + `Item` tables exist in schema. No endpoint, no UI.

### API — `packages/server/src/api/inventory.ts`

Export default `Router` (same pattern as `auth.ts` — not a factory function):

**`GET /api/inventory/:characterId`**
- Requires `requireAuth` only (no role check — GMs legitimately need to view player inventories; this is intentional).
- Returns the inventory for the given `characterId`:
```json
[
  {
    "id": "...",
    "quantity": 2,
    "item": { "id": "...", "name": "Espada Longa", "type": "weapon", "rarity": "rare", "stats": {} }
  }
]
```

**No no-param `/api/inventory` route.** `GamePage` always knows the `characterId` from `useAuth()` hook (`characterId` field on the JWT payload). Use `GET /api/inventory/:characterId` passing `characterId` from `useAuth`.

**`app.ts` registration:**
```ts
import inventoryRouter from './api/inventory'
// ...
app.use('/api/inventory', inventoryRouter)
```

### UI — `packages/client/src/components/game/InventoryPanel.tsx`

Props: `{ characterId: string; token: string; open: boolean; onClose: () => void }`

- Fetches `GET /api/inventory/:characterId` on first open (lazy, not on mount).
- Floating panel, `position: absolute`, bottom-left of game canvas, 260 px wide, max-height 380 px, `overflow-y: auto`.
- Styled with existing `index.css` classes: `.card`, `.badge`, `.section-title`, `.panel-section`.
- Rarity badge colors: common = `var(--parchment-dim)`, uncommon = `#5a9a5a`, rare = `var(--ember)`, epic = `#9966cc`, legendary = `#e8c820`.
- Empty state: "Nenhum item" in parchment-dim.
- Loading state: "Carregando..." text.
- Close button top-right (`×` in `.btn-ghost`).

**`GamePage.tsx` changes:**
- `const { token, characterId } = useAuth()`
- `const [inventoryOpen, setInventoryOpen] = useState(false)`
- In `handleKeyDown` (existing), add: `if (e.key === 'i' || e.key === 'I') setInventoryOpen(v => !v)`
- Render `<InventoryPanel characterId={characterId!} token={token!} open={inventoryOpen} onClose={() => setInventoryOpen(false)} />` inside the `position: relative` wrapper.

## Problem 4 — Spell Effects

### Root Cause Clarification
`createPhaserScene` itself works — it creates a Phaser game on an overlay canvas. `scene.add.graphics()` also works. The specific breakage is `scene.add.particles(x, y, undefined, config)` — Phaser 3.60+ requires a valid texture key; `undefined` throws. **Only `explosion` is broken.** `lightning` and `healing` use only `Graphics` and work correctly once a Phaser scene is created.

### Fix — `SpellFX.ts` `explosion` method

Replace `scene.add.particles(...)` with a `Graphics`-based expanding rings approach, matching the timing pattern of `lightning` (37 frames × 16 ms ≈ 600 ms):

```ts
async explosion(worldX: number, worldY: number, camera: Camera, radius = 64): Promise<void> {
  const { x, y } = camera.worldToScreen(worldX, worldY)
  const { game, scene, canvas } = await this.createPhaserScene(camera.viewW, camera.viewH)
  const gfx = scene.add.graphics()
  const FRAMES = 45

  scene.time.addEvent({
    delay: 16, repeat: FRAMES - 1, callback: () => {
      gfx.clear()
      const progress = frame / FRAMES  // 0→1
      // outer ring expands and fades
      const r1 = radius * progress
      const alpha1 = 1 - progress
      gfx.lineStyle(3, 0xff6600, alpha1)
      gfx.strokeCircle(x, y, r1)
      // inner ring, half size, slight offset
      if (progress < 0.6) {
        const r2 = radius * 0.5 * (progress / 0.6)
        gfx.lineStyle(2, 0xffdd00, (1 - progress / 0.6) * 0.8)
        gfx.strokeCircle(x, y, r2)
      }
      // sparks: 8 lines shooting outward
      for (let i = 0; i < 8; i++) {
        const angle = (i / 8) * Math.PI * 2
        const len = radius * 0.7 * progress
        gfx.lineStyle(1, 0xff9900, alpha1 * 0.7)
        gfx.beginPath()
        gfx.moveTo(x, y)
        gfx.lineTo(x + Math.cos(angle) * len, y + Math.sin(angle) * len)
        gfx.strokePath()
      }
      frame++
    }
  })
  let frame = 0
  this.cleanup(game, canvas, FRAMES * 16 + 100)
}
```
